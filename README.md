# Manual Testing Project – SauceDemo

## Project Overview
This project demonstrates manual testing of the SauceDemo web application. The project includes functional testing, negative testing, UI testing, validation testing, session testing, test case documentation, and bug reporting.

## Application Under Test
https://www.saucedemo.com/

---

## Modules Tested
- Login Module
- Products Module
- Cart Module
- Checkout Module

---

## Testing Types Performed
- Functional Testing
- Negative Testing
- UI Testing
- Boundary Testing
- Session Testing

---

## Test Summary
- Total Test Cases Executed: 24
- Passed Test Cases: 18
- Failed Test Cases: 6
- Bugs Identified: 9

---

## Bugs Identified
1. Login error message UI clipping issue
2. First Name field accepts numeric values
3. ZIP/Postal Code field accepts alphabetic values
4. First Name field accepts single character input
5. ZIP/Postal Code field accepts excessive length input
6. Last Name field accepts numeric values
7. Last Name field accepts single character input
8. Cart items persist across different user sessions
9. Validation weaknesses in checkout form

## Tools Used
- Manual Testing
- MS Excel
- GitHub Documentation

## Conclusion
The application is functionally stable. However, several validation, UI, and session-related issues were identified during exploratory testing.
